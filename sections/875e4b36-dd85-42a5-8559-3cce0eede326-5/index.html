Peritext's javascript modules

Peritext is built according to a modular logic that allows to assemble a specific writing and editing environment for specific (types of) research or publishing projects.
Different combinations can thus be set up according to the targeted media, the types of materials used in the documents or the type of publication envisaged (report, review, essay, etc.) and the most appropriate form of presentation.
Templates, contextualizers, and generators form the three types of building blocks that can be assembled to build a specific editorial environment with Peritext.

Contextualizers modules

In peritext, each resource can be displayed in content by means of contextualizations that define a set of parameters. For example, a resource of type "html code" can be successively contextualized as an embed contextualization, a source code contextualization, or a bibliographic reference.
In that frame, contextualizer modules define how to display a given resource type and expose a set of possible parameters for each mention.
For example, the "video" contextualizer allows you to display a "video" resource in the form of a video block, a thumbnail image (in print editions), or an audio excerpt in the body of a paragraph. It can be given the following parameters: start timecode, end timecode, volume level, interface display, ...
In addition, each contextualizer can produce differentiated renderings for the same contextualization according to two criteria which are the type of rendering support (paginated or on-screen) and the location of the contextualization: in the middle of a text block or stand-alone.

Templates modules

Templates are specific to a type of medium (web or print) and define both a set of graphic choices for formatting editions and the "blocks" that can be mobilized to compose the table of contents of a particular section.

Current templates

Deucalion

Deucalion is a web template for long reading oriented on the alternation between sequential and associative reading via the mention of materials in the contents. It proposes composition blocks to display the contents of the materials but also experimental blocks (geographical map, chronological frieze, network of mentions, etc.).

Pyrrah

Pyrrah is a print-oriented template designed for editing finalized texts. It proposes composition blocks corresponding to the conventions of print publishing (first and back cover, colophon, glossary, table of contents, bibliography, etc.).

Chrysaor

It is a template centered on hypertextuality and associative reading. It is based on a multi-columns layout inspired from one of the interfaces of the An inquiry into modes of existence project.

Calliroeh (work in progress)

This is a "research qualitative notebook" oriented template designed to allow the sharing of a journal or workspace with materials under analysis. All contributions are welcome!

Generators modules

Generators allow to transform an edition into a set of files that can be used for publication. Currently, generators are no longer used in the Ovide editor as external modules and integrated directly into the software. They may become necessary if new types of output formats are to be developed, such as:
  • IDML (indesign) files
  • reflowable epubs
  • fixed epubs
