Peritext's data model

Peritext relies on a common data model aimed at describing and manipulate scientific and editorial projects and their various publications, in the form of a data abstraction called "production".
The objective of the "production" model is to provide a way of describing writing and publishing projects that is adaptable to a variety of research materials, editorial contexts and media (including web and print), and that combines automation capabilities with the possibility for manual specification by publishers, designers and authors.
A Peritext output is a set of collections of information objects, the most important of which are of three types: resources, contextualizations, and editions. Let us know present the main features of these in a language accessible to non-technical people.

Design principles

Resources

In Peritext, a resource designates any element convened within the framework of an investigation or a writing project: bibliographic reference, glossary entry, video, image... the list is by definition extensible and adaptable to each writing and research context.
Each resource is associated with:
  • metadata (title, authors, dates, ...), described in a homogenous way, which makes it possible to meet a series of standards specific to scientific publishing in terms of indexing and citability.
  • data specific to its type (examples: URL for a video, geographic coordinates for a "location" type glossary entry, source code for a code resource)
  • annotations in the form of textual content.
In this framework, "section" type resources correspond to element which are not associated with any specific data and which may become, at the time of publishing chapters, parts, sub-parts, etc. Thus each resource can be the place where content is written and in which other resources are "contextualized".
The Peritext model is thus a "flat" model that does not presuppose a hierarchy of importance between "textual" and "non-textual" elements and allows a multitude of relationships mediated by "contextualization" parameters.

Contextualizations and contextualizers

Peritext is based on a mode of content representation that separates the resources called for in a production and the different forms that the latter may take in the context of argumentation operated by a text. These formatting modes can be specified very precisely according to the authors' rhetorical intentions, and reused in several places for complex formatting modes (example: multiple data visualizations with the same parameters allowing a comparison between several files).
When the contents of a resource are brought to "contextualize" another resource (for example: via a bibliographic citation), the relationship is then mediated by two types of objects:
  • a contextualizer, which is a set of parameters specifying how the source resource should be displayed in the contents of the target resource (e.g. by specifying a page number) and potentially differentiating between different output formats (web or print)
  • a contextualization, which is an object that acts as a junction between the target resource identifier, the source resource identifier, and the contextualizer's set of parameters used to establish the relationship.
This model allows great finesse in the way resources are summoned within content and allows for more "rhetorical possibilities" as to how to write with them, while maintaining great documentary integrity and avoiding redundant information.
The Peritext model thus makes it possible to combine documentary rigor and expressive power in the "contextualization" choices made by authors in multi-media publishing contexts.

Editions

Peritext is inspired by publishing workflow technologie (Crozat, 2012) and other single-source publishing initiatives that produce a variety of documents from the same source content. However, the intention of Peritext is to enable the production of a diversity of editions and to allow "automatic" production of editions for web and print as well as manual end-to-end specification and design.
To do this, a Peritext production contains a set of objects called editions. An edition is specific to a given medium (web or print) and contains:
  • a composition of contents, which can be resource contents or specific views (glossary, bibliography, table of contents, but also advanced views such as a geographical map, graph of the relations between resources,...)
  • specific CSS styles that allow to precisely specify the graphical (and interactive) aspect of each edition
  • a set of general parameters such as publication title, date, bibliographic citation style, ...
Thus, Peritext allows a large margin of freedom in the modes of materialization of the contents of a production according to the supports and the contexts of publication.
